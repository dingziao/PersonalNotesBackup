Registry

registry - repository - tag/digest

registry 是镜像仓库服务，包含多个repository。Docker Hub、AWS ECR

Docker Hub是默认的repository，FROM ubuntu会默认去docker.io/library/ubuntu 拉取。

repository 同一类镜像的不同版本。不同 tag/digest。

ECR: 存储image

ECS: image -> container

A container is a running instance of an image, including the image and its runtime state and files.

image不一定有tag; 一定有hash digest。

Docker命令

以下是将 app.py 打包成 Docker 镜像并运行的步骤：

1. 创建 Dockerfile：在与 app.py 同目录下创建一个名为 Dockerfile 的文件。

2. 创建 requirements.txt，列出所有依赖。

3. 构建 Docker 镜像：在当前目录中构建 Docker 镜像：docker build -t app-image .

t app-image：给镜像取名为 app-image。.：表示当前目录是构建上下文。

4. 运行 Docker 容器

应用在 Docker 容器中运行时，它感知到的 IP 是容器的 IP，通常是 Docker 网络中的一个局域网 IP，而不是主机的 IP。

启动容器并绑定端口（假设 app.py 运行在端口 5000）：

docker run -d -p 5000:5000 --name app-container app-image

d：后台运行容器。

p 5000:5000：将容器的端口 5000 映射到主机的端口 5000。

-name app-container：容器名称为 app-container。

app-image：使用之前构建的镜像。

docker rm app-container
docker rmi app-image

命令分类：

命令效果
构造时：主机FROM, COPY构造临时容器

运行时ENTRYPOINT, CMD运行时调用
FROM <基础镜像>
FROM python:3.13-slim (官方 Python Docker 镜像)
FROM AMAZON_LINUX

COPY <主机源> <镜像目的地>

RUN 

WORKDIR /app
// WORKDIR指定当前路径".",绝对路径不受影响。
COPY lib ./lib/ == COPY lib /app/lib/

ENTRYPOINT容器启动时的固定命令。不被覆盖。
CMD容器启动时的命令。可被覆盖。
//use case
ENTRYPOINT ["echo"]
CMD ["Hello"]
// docker run image -> Hello
// docker run image World -> World
