Go 并发基本：go func()则在新的goroutine 中异步执行，调用方不等待。

func main() {
    ch := make(chan string)
    go func() {
        ch <- "hello from goroutine"
    }()
	    msg := <-ch //会阻塞main，直到ch拿到消息。
    fmt.Println(msg)
}

channel、lock（mutex）、cond（条件变量）就构成了 Go 并发的核心结构。

持有锁=能跑

  • 类似一个有容量，会阻塞的消息队列
  • 显式锁，只能一人拿到
  • Wait() 会释放锁并阻塞，直到被 Signal() 或 Broadcast() 唤醒，然后重新竞争锁。
    算是 Lock() 和 Unlock() 之间的一个特殊中间态。
